# AI in Software Engineering

[← Software Engineering](what-is-software-engineering.md) | [Next: Market Usage](ai-usage-patterns.md)

This section describes how AI and software engineering meet in practice.

## Complementarity

AI excels at:
- generating code quickly
- explaining unfamiliar code
- proposing transformations
- accelerating repetitive work

Software engineering provides:
- structure and constraints
- validation and testing
- long-term coherence
- team-level discipline

## Tension

- AI is stateless; engineering requires continuity
- AI is probabilistic; engineering requires reliability
- AI is pattern-based; engineering requires explicit rules

## Working Model Observed

```text
human provides structure and intent
AI produces and transforms artifacts
human validates and aligns with system constraints
```
